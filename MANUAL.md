## 1. システム全体の流れ

```
① リミーが管理画面で企業を登録 → 専用URLが発行される
        ↓
② 企業の担当者が新入社員に専用URLを送る
        ↓
③ 新入社員がフォームに回答・送信（約5〜10分）
        ↓
④ AIが自動でレポートを生成（約1〜2分）
        ↓
⑤ メールが3通、自動送信される
   ・本人へ：パーソナルレポート
   ・マネージャーへ：マネジメントレポート
   ・リミー管理者へ：両方のレポート＋スコア表
        ↓
⑥ データがGoogleスプレッドシートに自動保存
        ↓
⑦ 毎月1日・15日に月次レポートを企業へ自動送信
```

## 2. 管理画面の使い方

### アクセス方法
`admin.html` をブラウザで開く → パスワードを入力

> パスワードはスクリプトプロパティの `ADMIN_SECRET` に設定されている値です。

---

### 企業を新規登録する

1. 管理画面の「企業登録」欄に入力：
   - **企業名**（例：株式会社サンプル）
   - **マネージャーのメールアドレス**（レポートの送付先）
   - **HRのメールアドレス**（任意。月次レポートの追加送付先）
2. 「登録」ボタンを押す
3. `LIMEE_XXXXXX` 形式の企業コードが発行される
4. 専用URLが表示される（コピーして企業担当者に渡す）

**専用URLの形式：**
```
index.html?code=LIMEE_XXXXXX&c=企業名
```

---

### 企業を削除する

管理画面の企業一覧から「削除」ボタンを押す。
※削除しても過去の回答データはスプレッドシートに残ります。

---

### 回答一覧を確認する

管理画面下部の「回答一覧」セクションで全企業の回答を確認できます。
スコアに応じてリスクバッジ（🔴高リスク / 🟡注意 / 🟢良好）が表示されます。

## 3. 企業への案内方法

### 企業担当者に伝えること

1. 専用URL（admin.htmlで発行）を送る
2. 以下の案内文を添える：

---

**案内文テンプレート：**

> ○○様
>
> お世話になっております。リミー事務局です。
> 入社90日サポート診断のご案内をお送りします。
>
> ▼新入社員の方に以下のURLをご共有ください
> 【専用URL】https://（ファイルのURL）?code=LIMEE_XXXXXX
>
> ・回答時間の目安：5〜10分
> ・回答後、本人とご担当マネージャーに自動でレポートが届きます
> ・ご不明点はこちらまで：miho.shinohe@remenow.com

---

### URLを渡すタイミングの目安
- 入社後 **2〜4週間目**が最適です
- 入社直後（1週間以内）は情報が少なく精度が下がります

## 4. 回答データの確認方法

### Googleスプレッドシートで見る

スプレッドシートには2つのシートがあります：

| シート名 | 内容 |
|----------|------|
| `responses` | 全回答データ（スコア・レポート・日時） |
| `companies` | 登録済み企業一覧 |
| `monthly_reports` | 月次レポートの送信ログ |

**responses シートの列構成：**
timestamp / company_code / company_name / name / email / gender / age / job / type / score_a〜e / employee_report / manager_report / result_id

## 5. 月次レポートについて

### 送信スケジュール
| 実行日 | 対象期間 | 件名例 |
|--------|----------|--------|
| 毎月1日 | 前月16日〜前月末日 | ○月 後半 オンボーディングレポート |
| 毎月15日 | 当月1日〜当月14日 | ○月 前半 オンボーディングレポート |

### 送信先
- 各企業のマネージャーメールアドレス
- 各企業のHRメールアドレス（登録がある場合）
- リミー管理者（BCC）

### 対象期間に回答がなかった場合
その企業へのレポートは送信されません（スキップされます）。

### 手動で送信テストしたい場合
Google Apps Script エディタで `sendMonthlyReports` 関数を選択して「実行」ボタンを押す。

## 6. よくあるトラブルと対処法

### 「URLを開いたらエラーが出る」
- URLに `?code=LIMEE_XXXXXX` が含まれているか確認
- 企業コードが管理画面に登録されているか確認

### 「送信したのにメールが届かない」
- Gmailの迷惑メールフォルダを確認するよう案内
- スプレッドシートにデータが保存されていれば、送信処理自体は完了している
- Apps Scriptの実行ログで確認：エディタ →「実行数」メニュー

### 「グルグル（ローディング）が終わらない」
- Claude APIの処理に通常1〜2分かかります。3分以上続く場合はネットワークエラーの可能性があります
- ページを閉じて再度回答しても、スプレッドシートに保存される前に止まっていれば問題ありません

### 「Apps Scriptのエラーが出た」
- エディタ左メニュー「実行数」でエラー内容を確認
- よくある原因：APIキーの期限切れ → Anthropicコンソールで確認・再発行

## 7. 設定を変更したいとき

### 専門家の名前・URLを変更する
Apps Script エディタで `getConfig()` 関数内の以下を編集：
```javascript
pharmacistName: '薬剤師 ○○ ○○',
pharmacistUrl:  'https://（相談URL）',
ptName:         '理学療法士 ○○ ○○',
ptUrl:          'https://（相談URL）',
```
変更後は**再デプロイが必要**です（新しいバージョンで更新）。

### AIプロンプトを変更する
同じくApps Script内の `callClaudeForEmployee()` または `callClaudeForManager()` 関数を編集。
変更後は再デプロイが必要です。

### 再デプロイの手順
1. Apps Script エディタを開く
2. 「デプロイ」→「デプロイを管理」
3. 鉛筆アイコン（編集）→ バージョンを「新しいバージョン」に変更
4. 「デプロイ」ボタンを押す

> ⚠️ 再デプロイしてもURLは変わりません。app.js や result.html の書き換えは不要です。

## 8. システム構成メモ（引き継ぎ用）

### ファイル保存場所
`/Users/mihomatuo/onboarding-diagnosis/`

| ファイル | 役割 |
|----------|------|
| `index.html` | 診断フォーム（新入社員が開くページ） |
| `result.html` | 結果閲覧ページ（URLシェア用） |
| `admin.html` | 管理画面 |
| `app.js` | フロントエンドのロジック |
| `style.css` | デザイン |
| `apps-script.gs` | バックエンド（Google Apps Scriptに貼り付けて使う） |
| `privacy.html` | プライバシーポリシー |

### スクリプトプロパティ一覧
Google Apps Script エディタ →「プロジェクトの設定」→「スクリプトプロパティ」で確認・編集できます。

| キー | 内容 |
|------|------|
| `SPREADSHEET_ID` | GoogleスプレッドシートのID |
| `CLAUDE_API_KEY` | Anthropic APIキー |
| `ADMIN_SECRET` | 管理画面のパスワード |
| `SITE_URL` | サイトのURL（ホスティング後に設定。例：https://xxx.github.io/onboarding-diagnosis） |

### 使用サービスのアカウント
- **Anthropic（Claude API）**：APIキーの管理・請求
- **Google Apps Script**：バックエンド・メール送信・スプレッドシート
- **Googleスプレッドシート**：データ保存
